LLM-based Domain Mapper

Semantic translation between financial and healthcare records using large language models.

This project explores how large language models (LLMs) like GPT-4 can be used to semantically map data from one domain (e.g., healthcare) to another (e.g., finance), preserving meaning while rephrasing structure and context.

The core of the system uses an encoder-decoder pipeline where raw tabular or JSON data is first converted into a semantic embedding. An LLM is then prompted with structured instructions to output the same data conceptually, but framed using the ontology and terminology of the target domain.

Use Case Example

In this demo, we mapped a medical record describing a patient’s treatment history into a format used by insurance claims processors. The LLM successfully translated ICD codes into claim billing categories, patient summaries into premium-impact narratives, and timelines into structured event logs.

How it Works


// Pseudo-prompt sent to the LLM

"Input: {
  'diagnosis': 'Hypertension',
  'procedure': 'ECG',
  'date': '2023-04-01'
}

Task: Translate this into an insurance claim format using financial domain terminology."

The model responded with: 

{
  "service_type": "Cardiology",
  "billing_code": "99284",
  "diagnosis_mapped": "High blood pressure",
  "claim_date": "2023-04-01"
}

We also implemented a feedback loop using vector similarity scoring to validate the accuracy of mappings based on both domain-specific knowledge bases and few-shot learning examples.
